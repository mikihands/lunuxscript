## 低リソース環境向けブルー‑グリーンデプロイスクリプト

このスクリプトは、CPU・RAM が限られたサーバー（例：Raspberry Pi、 小規模クラウド VM）でも安定した **ブルー‑グリーンデプロイ** を実現できるよう最適化されています。

### 主な機能
- **リソース最適化**  
  新しい環境を起動する前に、旧環境で CPU を大量に消費するバックグラウンドワーカー（Celery）を戦略的に停止し、CPU スパイクを防ぎます。

- **ヘルスチェック**  
  トラフィックの切り替え前に、新環境の状態を自動的に検証します。

- **グレースフルロールバック**  
  新環 বক্তব্যがヘルスチェックに失敗した場合、スクリプトが自動的に旧環境のバックグラウンドサービスを復元します。

- **手動検証**  
  「Human‑in‑the‑loop」ステップを組み込み、管理者がデプロイを最終確認し、古いコンテナを永久に削除する前に検証できるようにしています।

- **プロジェクト分離**  
  Docker Compose のプロジェクト名（`-p`）を使用して、同一の環境を2つ並行して実行します。
