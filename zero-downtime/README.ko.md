## 저자원 환경을 위한 블루‑그린 배포 스크립트

이 스크립트는 CPU·RAM이 제한된 서버(예: 라즈베리 파이, 소형 클라우드 VM)에서도 안정적인 **블루‑그린 배포** 방식을 제공하도록 최적화되었습니다.

### 주요 기능
- **리소스 최적화**  
  새 환경을 시작하기 전에 기존 환경의 CPU를 많이 잡아먹는 백그라운드 워커(Celery)를 전략적으로 중지하여 CPU 급증을 방지합니다.

- **헬스 체크**  
  트래픽을 전환하기 전에 새 환경의 상태를 자동으로 검증합니다.

- **우아한 롤백**  
  새 환경이 헬스 체크에 실패하면 스크립트가 자동으로 이전 환경의 백그라운드 서비스를 복구합니다.

- **수동 검증**  
  “Human‑in‑the‑loop” 단계가 포함돼 있어 관리자가 배포를 최종 확인하고 오래된 컨테이너를 영구적으로 삭제하기 전에 검토할 수 있습니다.

- **프로젝트 격리**  
  Docker Compose 프로젝트 이름(`-p`)을 사용해 두 개의 동일한 환경을 나란히 실행합니다.
